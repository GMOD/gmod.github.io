# <span id="Managing_Next_Generation_Sequence_Data_with_GMOD" class="mw-headline">Managing Next Generation Sequence Data with GMOD</span>

Poster by [Dave Clements](User:Clements "User:Clements"), [Scott
Cain](User:Scott "User:Scott"), Paul Hohenlohe, Nicholas Stiffler, Paul
Etter, Eric Johnson, and William Cresko, presented at [PAG
2009](PAG_2009 "PAG 2009"). This poster is also available
<File:PAG2009NextGenSeqPoster.pdf>.

## <span id="Abstract" class="mw-headline">Abstract</span>

Next generation sequencing is flooding many organizations with enormous
amounts of genomic data. A single machine can now produce three billion
base pair reads (the size of the human genome) every three days.
Components from the GMOD Project
(<a href="http://gmod.org" class="external free"
rel="nofollow">http://gmod.org</a>) can help visualize, manage and
annotate this deluge of data. We describe how to use GMOD tools to gain
new insights with high-throughput sequence data.

## <span id="Content" class="mw-headline">Content</span>

Poster describes using high throughput sequencing with threespine
stickleback populations and then visualizing the resulting allele and
genotype frequencies (SNPs) using [GBrowse](GBrowse.1 "GBrowse"). Also
discusses [Apollo](Apollo.1 "Apollo") and
<a href="Chado" class="mw-redirect" title="Chado">Chado</a>.
